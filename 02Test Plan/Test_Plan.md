# Test Plan Document

#Project Name: OpenCart Demo E-Commerce Website  
#Project Type: Web Application  
#Testing Type: Manual Functional Testing  
#Prepared By: QA Tester  

---

# 1. Introduction

This Test Plan describes the testing strategy, scope, approach, and resources required for testing the OpenCart demo e-commerce website. The purpose is to ensure the application functions correctly for end users before release.

---

# 2. Objectives

- Validate application functionality as per requirements  
- Identify defects before release  
- Ensure smooth user shopping experience  
- Verify core modules like registration, login, cart, and checkout  

---

# 3. Scope

# In Scope
- User Registration  
- User Login  
- Product Search  
- Product Details Page  
- Add to Cart  
- Cart Management  
- Checkout Process  
- Order Confirmation  
- Logout & Session Handling  

# Out of Scope
- Performance Testing  
- Security Penetration Testing  
- Payment gateway backend validation  
- Mobile app testing  

---

# 4. Test Types

| Test Type          | Applicable   |
|--------------------|------------  |
| Functional Testing | Yes          |
| UI Testing         | Yes          |
| Regression Testing | Yes          |
| Usability Testing  | Basic        |
| Performance Testing|  No          |

---

# 5. Test Approach

- Testing will be performed manually  
- Test Scenarios will be derived from requirement analysis  
- Detailed test cases will be created for each scenario  
- Bugs will be logged with screenshots  
- Regression testing will be performed after bug fixes  

---

# 6. Test Environment

| Item | Details |
|------|---------|
| Application | OpenCart Demo Website |
| Browser | Google Chrome |
| Operating System | Windows 10 |
| Testing Type | Manual |

---

# 7. Entry Criteria

Testing will start when:

- Application is accessible  
- Requirements are available  
- Test scenarios and test cases are prepared  

---

# 8. Exit Criteria

Testing will end when:

- All test cases are executed  
- No critical defects remain open  
- Test Summary Report is prepared  

---

# 9. Roles & Responsibilities

| Role          | Responsibility                        |
|---------------|---------------------------------------|
| QA Tester     | Test design, execution, bug reporting |
| Developer     | Fix defects                           |
| Product Owner | Requirement clarification             |

---

# 10. Risks & Mitigation

| Risk                 | Mitigation                           |
|----------------------|--------------------------------------|
| Application downtime | Inform team and resume testing later |
| Requirement changes  | Update test cases accordingly        |
| Browser issues       | Test in supported browser            |

---

# 11. Deliverables

- Requirement Analysis Document  
- Test Plan  
- Test Scenarios  
- Test Cases  
- Bug Reports  
- Test Execution Report  
- Test Summary Report  

---

# 12. Conclusion

This test plan ensures systematic testing of the OpenCart e-commerce workflow to deliver a stable and reliable application to end users.
